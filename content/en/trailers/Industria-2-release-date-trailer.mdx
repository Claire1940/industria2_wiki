## Platform and Performance Planning Before Day One

You don’t need final benchmark charts to prepare intelligently. In 2026, most launch-week issues are solved by pre-configuration and disciplined settings management.

### Quick performance preparation matrix

| System Area | Baseline Recommendation | If You Have Issues |
|---|---|---|
| **Storage** | Install on SSD/NVMe | Move from HDD to SSD first |
| **RAM** | Keep background apps minimal | Close browser tabs, launchers, overlays |
| **GPU settings** | Start at Medium preset | Lower shadows and volumetrics first |
| **Display mode** | Exclusive fullscreen | Switch from borderless if stutter appears |
| **Frame target** | Lock to stable range | 60 locked often feels better than unstable 90+ |

### Day-one settings order (fast method)

| Step | Action | Time Cost |
|---|---|---|
| **1** | Launch with default preset once | 2 minutes |
| **2** | Run first scene, check frame pacing | 3 minutes |
| **3** | Tune shadows, post-processing, fog | 5 minutes |
| **4** | Lock frame cap + enable upscaling if needed | 3 minutes |
| **5** | Restart game after final changes | 1 minute |

This process prevents the common loop of random toggling and inconsistent results. If you want a smooth first impression, follow an order, test in one area, and change only one variable at a time.
